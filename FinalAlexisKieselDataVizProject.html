Alexis' Data Visualization Project: Leaflet

Leaflet is the leading open-source JavaScript library for mobile-friendly interactive maps. Leaflet can be used in many ways. You have most likely seen Leaflet used without even realizing it. It is widely used, especially in building web mapping applications. This tool is useful especially in making interactive maps. It has all the mapping features and tools that most developers might need. It has many plugins. First being released in 2011, Leaflet is supported by most mobile and desktop platforms and can be used with HTML5 and CSS3. It is one of the most popular JavaScript mapping libraries along with OpenLayers and the Google Maps API. It can be seen in use by major websites such as Pinterest, Flickr and FourSquare. Leaflet allows developers who lack a GIS background to make tiled web maps hosted on a public server with ease. GeoJSON files, which is what I used along with my html document, can be loaded into leaflet and can create and style interactive layers. The tool was developed by Vladimir Agafonkin.

Limitations of the tool

The maps displayed on Leaflet are based on OpenStreetMap, which is a free editable map of the world. This means the map could be inaccurate considering it can be edited by any Internet user. Another limitation of Leaflet is that it is simply a JavaScript library. It is free, but it does not provide maps. One must choose a tile service to combine with it. I have also read that many people have had issues with memory leaks using Leaflet.
