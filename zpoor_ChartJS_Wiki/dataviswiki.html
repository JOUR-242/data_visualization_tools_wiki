Wait, what is it?

Chart.js is an open source JavaScript library that creates simple with charts with a modern, flat design. Designers and developers can quickly create six different types of core charts (bar, line, radar, pie, doughnut and polar area) with the basic script file.

Ok, sounds cool. Where can I get it?

Everything Chart.js can be found on its website, chartjs.org. The script file download, as well as sample HTML documents, instructions and other helpful information, can all be found on GitHub. The Chart.js library is the only thing download necessary to use this tool, besides some kind of text editor. Chart.js uses the canvas element of HTML5 to build charts on the web.

What have people done with this tool?

Chart.js is a rather basic tool, an excellent way to quickly get visualized data online. However, because of its basic nature, many data journalists look to more expansive tools to best show their data and tell a larger story. Below are two examples of journalistic pieces that utilize programmatic charts with similar interactive features as offered with Chart.js.

"Nursing Home Inspect" by ProPublica

This data visualization from ProPublica combines two chart types-- one of which Chart.js can create, one of which it cannot, to best tell this story. Chart.js can create the bar charts shown, but not the corresponding maps. This visualization explores data surrounding problems reported in nursing homes across the country. To do this, it uses a set of maps and corresponding bar charts. It uses one of the main interactive features of Chart.js--hover--to explore this data. Upon hovering over a specific state or bar, it's corresponding match will also highlight, to make the data more accessible and understandable.

"Explore the Data" by The New York Times

This data visualization from The New York Times explores monetary incentives given to businesses by local and state governments. To interact with this chart, users hover over a state to see the preliminary data of total money received. This feature is similar to what can be accomplished with a Chart.js chart, despite that fact that Chart.js cannot create maps. To further explore the data, users click on the state so see individual businesses in that state and the amounts of money they were given.

How on earth do I use this thing?

To show you how easy it is to use this tool, I'll walk through a simple example of how to make the basic bar chart below. Each of the six available chart types use very similar steps and code, all of which can be found on GitHub.

1. First, you need to add the appropriate script file to your HTML document in the head tag. The Chart.js file can be downloaded from GitHub.

2. Chart.js makes use of the canvas element in HTML5. In order to display your chart on your HTML page, you need to create a div that can hold the canvas element. It is important to give the canvas element a specific id, as this value will be necessary to render the chart later.

3. Once the div and canvas are set up, you can being writing your JavaScript in a script tag. You want to create a variable that will hold all the data for your chart. The code and documentation that can be downloaded from GitHub has example data variables for each of the six available chart types that can be copy and pasted and then have the data swapped out. The labels set allow you do declare the number of data points your chart will have as well as label them, in this example they are months. The datasets allows you to enter the specific data as well as format the chart. In this example, the bars' fill color, stroke color, highlight fill, highlight stroke as well as that actual data were determined within the "datasets." Multiple data sets can be entered allowing for, in this example, multiple bars to appear for each specified month.

4. Finally you can run create your chart by running the function. The window.onload function is one already created in the Chart.js library and creates a chart by locating the canvas element and rendering a new chart using the data entered in the dataset variable. Closing the script tag will end the use of JavaScript in the document.

5. When the HTML file and Chart.js are uploaded to a hosting site, the chart will run on that page. This version of Chart.js also has the ability to be responsive (you can turn this feature on or off using a Boolean operator in the window.onload function). The chart is automatically interactive- allowing for data points to be shown on hover- because of the built in functionality of the Chart.js library. So there you have it! In just a few easy steps, you can have a beautiful, interactive chart.

What can't this thing do?

Chart.js is great because it is a simple and easy-to-use tool that even the most novice designer or developer can use. However, to crate the ease of use, it is not as powerful as some other programmatic data visualization tools. The following list describes some of the limitations of Chart.js

  • Unable to include a legend for the chart axes
  • No zooming or scrolling function
  • Not able to have multiple axes or trendlines
  • Only six chart types; not able to create horizontal bar charts, stacked charts, scatter charts or others

Chart.js allows for quick and easy chartmaking on the web, with the sacrifice of some advanced features in exchange for speed and ease.

Anything else I should know?

Chart.js is a pretty powerful tool that can do a lot of things. Don't stress out trying to memorize all the incredible things it can do. View the Chart.js documentation to look up everything about the tool.
